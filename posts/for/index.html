October 9, 2018

Для кого мы создаем сайты?!

  • Для людей

    UX/UI - дружелюбный, понятный, удобный и красивый сайт с предсказуемым поведением

    Сайт должен загружаться быстро и потреблять минимально возможный объем трафика.

  • Для поисковых роботов и программ, читающих вслух.

    Использование семантичных тегов HTML, помогающие алгоритмам поиска находить и индексировать только тот контент, который может быть полезным.

    Люди с ограниченными возможностями могут использоват программы - ридеры для чтения контента вслух и для навигации только клавиатурой или только голосом. Атрибуты area и role помогают в этом, а панель инструментов Accessibility Inspector показывает нам как выглядит описание страницы для программы - ридера.

  • Для себя/ для коллег

    Хорошо отформатированный, читабельный код с комментраиями, продуманной системой именования директорий (папок), файлов, классов, id. 

    

    
<p class="gold16">
    Добро пожаловать!
</p>

    
<p class="greeting">
    Добро пожаловать!
</p>
  • Для смежников

    Для дизайнера - максимально точно сверстать макет. Про непонятное - узнавать. Про спорное и лучшее - предлагать.

    Для бакенда - еще раз подумать о системе именований классов и верстать так, чтобы вместо контента легко можно было подставлять значения из базы данных: 

    

    
<span class="value">
    100грн
</span>

    
<span class="value">
    100
</span>
<span class="currency">
    Грн
</span>

  • Для заказчика

    Уметь размещять сайт на хостинге (linux + apache/nginx) - т.е. уметь копировать папки и файлы и избегать заглавных букв в названии. Знать об относительном и абсолютном пути.

    Знать и уметь применять все современные техники по оптимизации сайта - т.е. делать все вышеперечисленное. Уметь аргументированно возражать на неоптимальные предложения со стороны заказчика.
